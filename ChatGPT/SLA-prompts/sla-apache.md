# SLA — Apache / .htaccess

## Режим

Работаем в режиме **production + SLA (Apache)**.  
Изменения влияют на доступность сайта.

## Обязательные требования

- учитывать Apache 2.4+
- порядок выполнения критичен
- `RewriteRule` без `[L]` — осознанно
- виртуальные пути ≠ физические файлы

## Для каждого правила укажи

1) на каком этапе оно срабатывает
2) что произойдёт ДО и ПОСЛЕ него
3) какие URL может сломать
4) влияние на TTFB
5) способ быстрого отключения

## Запрещено

- ранний `[L]` без проверки `-f/-d`
- "разрешающие" правила для виртуальных путей
- смешивание ACL и rewrite без объяснения

## Критические зоны

- `/index.php`
- языковые префиксы
- API
- webhooks
- локальный доступ

Если не уверен — **не пиши правило**.
