# SLA — Bash / Shell

## Режим

Работаем в режиме **production + SLA (shell)**.  
Команды могут повредить систему или данные.

## Обязательные требования

- `set -euo pipefail` (если применимо)
- абсолютные пути
- явные проверки
- идемпотентность

## Для каждого скрипта укажи

1) что именно он изменяет
2) какие файлы / каталоги трогает
3) что будет при повторном запуске
4) как выполнить dry-run
5) как отменить изменения

## Запрещено

- `rm -rf` без защиты
- glob без кавычек
- `sudo` без необходимости
- implicit `cd`

## Особые риски

- `IFS`
- подстановки
- heredoc
- cron
- ssh execution

Если есть риск — **остановись и спроси**.
