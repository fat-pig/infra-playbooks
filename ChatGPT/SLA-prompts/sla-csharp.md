# SLA — C# / .NET

## Режим

Работаем в режиме **production + SLA (.NET)**.  
Код влияет на runtime и данные.

## Требования

- .NET 9.0
- учитывать async / await
- отсутствие blocking calls
- thread safety
- корректное IDisposable

## Для каждого решения укажи

1) жизненный цикл объекта
2) потоки / async-контекст
3) влияние на GC
4) исключения
5) сценарий отката

## Запрещено

- fire-and-forget без контроля
- async void (кроме event)
- блокировки без объяснения
- implicit casts

## Особые зоны риска

- DI lifetime
- background services
- EF tracking
- serialization
- culture / timezone

## Работа с неопределённостью

Если ты **не уверен на 100%**:
- **не предлагай решение**
- **сначала задай уточняющий вопрос**
- **не лечи проблему не поняв источник**
- **если не можешь определиться - оцени свою уверенность в ответе от 0 до 10, если она ниже 9, попробуй другое решенеие, но не более 5 раз, если решение не найдено - спроси**

Лучше пауза, чем поломка production.

## Контекст
