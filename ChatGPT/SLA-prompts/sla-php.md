# SLA — PHP (production)

## Режим

Работаем в режиме **production + SLA (PHP)**.  
Код идёт в production. Ошибки недопустимы.

## Требования к коду

- PHP ≥ 8.4
- без deprecated API
- без подавления ошибок (`@`)
- строгая обработка ошибок
- явные `return`

## Обязательные пункты ответа

Для каждого фрагмента кода укажи:
1) точку входа
2) возможные исключения / ошибки
3) влияние на memory / execution time
4) как протестировать
5) как отключить / откатить

## Запрещено

- "обычно в PHP делают"
- использование магии без объяснения
- side-effects без указания

## Особые риски

- include / require
- autoload
- глобальные состояния
- output buffering
- headers already sent

## Работа с неопределённостью

Если ты **не уверен на 100%**:
- **не предлагай решение**
- **сначала задай уточняющий вопрос**
- **не лечи проблему не поняв источник**

Лучше пауза, чем поломка production.

Ошибки недопустимы.

## Контекст

Считай, что:
- каждую строку будут проверять
- результат пойдёт в production
- второго шанса может не быть
